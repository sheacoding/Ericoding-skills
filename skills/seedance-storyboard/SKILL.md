# 🎬 Seedance 2.0 分镜提示词生成器

帮助用户将想法转化为专业的 Seedance 2.0 视频分镜提示词。

## Seedance 2.0 核心能力

### 技术参数

| 输入类型 | 格式要求 | 数量限制 | 大小限制 |
|---------|---------|---------|---------|
| **图片** | jpeg、png、webp、bmp、tiff、gif | ≤ 9 张 | < 30 MB |
| **视频** | mp4、mov | ≤ 3 个，总时长 2-15s | < 50 MB |
| **音频** | mp3、wav | ≤ 3 个，总时长 ≤ 15s | < 15 MB |
| **文本** | 自然语言 | - | - |

**混合输入总上限：12 个文件**

> 💡 建议：优先上传对画面或节奏影响最大的素材。有参考视频会消耗更多生成额度。

**核心功能**：

1. **参考图像** - 精准还原画面构图、角色细节、服装样式
2. **参考视频** - 支持镜头语言、复杂动作节奏、创意特效复刻
3. **视频延长** - 平滑延长与衔接，可"接着拍"
4. **视频编辑** - 角色更替、剧情颠覆、片段调整、场景融合
5. **多模态组合** - 图+视频+音频+文本自由组合创作

**基础能力增强**：
- ✅ 物理规律更合理、动作表现更自然流畅
- ✅ 指令理解更精准、风格保持更稳定
- ✅ 支持复杂动作、连续运动等高难度生成任务

**交互方式**：使用"@素材名"指定用途，如"@图片1 作为首帧，参考@视频1 的运镜"

**入口说明**：
- **首尾帧入口**：只上传首帧图 + prompt
- **全能参考入口**：多模态（图、视频、音频、文本）组合输入

### 第四步：内容细节收集

#### 3. 镜头语言
- 景别变化：远景→全景→中景→近景→特写？
- 运镜方式：推/拉/摇/移/跟/环绕/升降/希区柯克变焦？
- 转场方式：硬切/渐变/匹配剪辑/特效转场？

💡 **四段式运镜公式**（AI漫剧高频模式，可直接套用）：
```
开场(0-3s)：低角度跟拍/慢推 - 建立场景氛围
发展(4-8s)：环绕摇镜快切特写 - 展示动作细节
高潮(9-12s)：仰拍拉远定格慢放 - 情绪/特效爆发
收尾(13-15s)：缓推特写渐暗 - 意境落版
```

💡 **色调宣言开头**：用色调定义整个视频基调
- 示例："暖冷色调剧烈反差"、"紫黑冷色调"、"金红暖色调"
- 建议：放在提示词开头，影响整体画面风格

💡 **嵌入式对白**：将角色台词融入时间轴
- 示例：主角低喝 "破！"、女皇低吟 "这冰蚀之力…竟已入骨"
- 效果：让AI理解情绪节奏，生成更贴合的表情动作

#### 4. 动作与节奏
- 主体动作：行走/奔跑/打斗/舞蹈/表情变化？
- 动作节奏：舒缓/急促/有张有弛？
- 是否需要音乐卡点？

#### 5. 声音设计
- 配乐风格：激昂/温馨/悬疑/欢快/史诗？
- 音效需求：环境音/动作音效/特殊音效？
- 对白/旁白：有台词吗？什么语气？

💡 **逐拍声音设计**（高阶技法）：
- 为每个时间段独立设计音效
- 示例："伴随冰晶炸裂的脆响与低温嘶鸣"、"雷鸣滚滚配合剑刃破空声"
- 效果：让声音与画面节奏精准同步


#### 视觉张力原则（CRITICAL — 提升画面冲击力的核心）

**颜色对比**：两个视觉主体必须形成鲜明的色彩反差
- 金 VS 紫、红 VS 蓝、白 VS 黑、橙 VS 青
- 示例："金色灵力与紫色暗能量在空中碰撞"

**材质对比**：不同质感的视觉碰撞
- 金属 VS 布料、火焰 VS 冰霜、机械 VS 有机、流体 VS 固体
- 示例："鎏金铠甲表面映射出冰蓝色的寒光"

**动静对比**：节奏的张弛
- 慢动作蓄力 → 瞬间爆发、静默凝视 → 高速冲刺
- 示例："时间仿佛凝固，下一帧瞬间加速"

#### 动态描述词库（让 AI 理解物理模拟）

**破坏类动词**：
- 破碎 Shattering、粉碎 Crushing、崩裂 Cracking、瓦解 Disintegrating

**变化类动词**：
- 溶解 Dissolving、结晶 Crystallizing、蒸发 Evaporating、凝固 Solidifying

**运动类动词**：
- 流动 Flowing、涌动 Surging、旋转 Spiraling、扩散 Spreading

**能量类动词**：
- 迸射 Bursting、汇聚 Converging、脉冲 Pulsing、过载 Overloading

#### 逐镜特效描写规范

**每个镜头必须包含独立的【特效】块**，描述以下维度：
1. **粒子效果**：火星飞溅、冰晶漫天、光点消散、灰烬飘落
2. **流体模拟**：能量波纹扩散、液态金属流动、烟雾翻涌
3. **光影变化**：逆光剪影、体积光穿透、光效过载致白
4. **物理交互**：地面龟裂、碎石悬浮、冲击波推倒周围物体
5. **镜头效果**：镜头震动、画面定格、子弹时间 Bullet time、速度线

#### 镜头语言双语速查

| 中文 | English | 效果 |
|------|---------|------|
| 推镜头 | Zoom in / Push in | 强调主体，营造紧张 |
| 拉镜头 | Pull out / Zoom out | 展现全景，营造疏离 |
| 环绕镜头 | Orbit shot | 360度展示主体 |
| 慢动作 | Slow motion | 强调关键瞬间 |
| 子弹时间 | Bullet time | 极致慢放+环绕 |
| 手持晃动 | Handheld shake | 纪实感/紧张感 |
| 升格镜头 | High-speed / Overcrank | 超慢动作细节 |
| 低角度仰拍 | Low angle shot | 强调力量感/压迫感 |
| 俯冲镜头 | Dive shot | 从高处急速下降 |
| 甩镜头 | Whip pan | 快速转场/切换视角 |

**提示词书写原则**：
1. **格式声明优先**：竖屏优先声明"9:16竖屏"，横屏声明"16:9横屏"或"2.35:1电影宽屏"
2. **视觉张力优先**：确保画面中存在颜色对比、材质对比或动静对比
3. **逐镜特效**：每个时间段必须包含独立的特效描写，不能只写动作
4. **动态动词**：多用物理模拟词汇（破碎、溶解、结晶、涌动），避免"出现"、"变成"等静态词
5. 时间轴清晰，标注每段的秒数范围
6. 镜头语言明确（推/拉/摇/移/跟/环绕等）
7. 多模态引用时使用 **@素材名** 或 **"素材图X中的主体人物"** 格式（两种语法可互换）
8. 需要延长视频时说明"延长Xs"并选择对应生成长度

⚠️ **平台限制**：禁止使用具体人名，用"红衣剑客"、"白衣法师"、"素材图X中的人物"等通用描述

⚠️ **硬性字数限制**：生成的提示词正文（不含格式标签）**必须控制在 1000 字以内**。超出会导致平台截断或生成质量下降。宁可精炼描述，不可堆砌冗余。

### 第五步：优化与建议

- **字数检查**：确认提示词正文 ≤ 1000 字，超出则精简低优先级描述
- 检查是否违反平台规则（如写实人脸限制、具体人名）

## 高阶技法（实测验证）

### 品类专项指南

#### AI漫剧专项技法
**核心公式**：首尾帧变身 + 四段式运镜 + 特效密集描写

**首尾帧变身技法**：
```
首帧：@图片1 的A形态（如：红金战袍形态）
尾帧：@图片2 的B形态（如：冰霜形态）
过程：15秒变身镜头，从首帧A形态逐渐转变为尾帧B形态
```

⚠️ **平台限制提醒**：
- **禁止使用具体人名**（如天运子、暗金帝、李白等），会导致审核不通过
- **推荐用法**：通用角色描述（红衣剑客、白衣女子、老者、少女）、职业身份（法师、战士、商人）、形态描述（红金形态、冰霜形态）

**特效描写词库**：
- 能量类：灵力涌动、法阵展开、符文环绕、粒子爆发
- 元素类：冰晶炸裂、火焰缠绕、雷霆轰鸣、水波荡漾
- 光效类：金光迸射、紫光涌现、星芒闪耀、虚影重叠
- 材质类：鎏金质感、冰霜覆盖、熔岩纹理、星空粒子

**示例提示词结构**：
```
紫黑冷色调，15秒，首尾帧变身

0-3秒：低角度慢推，@图片1 红金战袍人物立于王座，周身金色灵力涌动
4-8秒：环绕摇镜，人物起身，手中凝聚冰蓝法阵，空气开始结霜
9-12秒：仰拍拉远，冰蚀之力爆发，身形被冰晶包裹，低吟"破！"
13-15秒：特写渐显，冰晶碎裂，@图片2 冰霜形态显现，定格

伴随冰晶炸裂的脆响与低温嘶鸣，背景史诗配乐
```

#### 真人短剧专项技法
**核心要素**：角色照片驱动 + 对白嵌入 + 场景真实感

**素材引用语法**：
- 标准语法：`@图片1 作为主角形象`
- 小云雀语法：`素材图1中的主体人物` （平台官方推荐）
- 两种可互换，后者更明确指向人物主体

**对白驱动结构**：
```
0-3秒：[场景建立] + [人物出场]
3-5秒：素材图1中的人物说："[台词]"，[表情/动作描述]
5-8秒：素材图2中的人物回应："[台词]"，[表情/动作描述]
8-12秒：[冲突/转折] + [情绪变化]
12-15秒：[结局] + [落版]
```

⚠️ **避免使用**："角色A"、"角色B"、"主角"等，推荐用"素材图X中的人物"或具体描述

**示例**：
```
9:16竖屏，真人短剧风格，15秒

0-3秒：中景，年夜饭餐桌，素材图1中的老者独自坐着，神情落寞
3-5秒：特写，老者叹气："又是一个人过年啊…"
5-8秒：门铃响起，老者惊讶抬头
8-12秒：门打开，素材图2中的年轻女孩抱着礼物冲进来："我回来啦！"
12-15秒：两人拥抱，老者笑中带泪

温馨背景音乐 + 门铃声 + 对白
```

⚠️ **用词规范**：
- 老者/老人（替代"爷爷"）
- 年轻女孩/少女（替代"孙女"）
- 中年男子/女子（替代"叔叔/阿姨"）

#### 互动接龙专项技法
**核心逻辑**：社交钩子 + 多集连续性 + 参与引导

**系列视频结构**：
- 第一集：建立规则 + 示范 + 抛出问题
- 第二集：承接上集 + 新内容 + 再次抛出
- 每集结尾：引导评论区参与

**社交钩子设计**：
```
【加一字系列】
第一集："我+加一字=找" → "你能接下去吗？"
第二集："找+加一字=找到了你" → "继续接龙！"

【成语接龙】
第一集："一马当先" → "先下手为强"
第二集："强颜欢笑" → "下一个成语是？"
```

**提示词要点**：
- 明确标注"第X集"或"接上集内容"
- 保持角色/场景一致性
- 结尾留互动钩子

#### P视频/视频编辑专项技法
**核心指令**：继承 + 修改 + 保留声明

**视频编辑指令速查**：

## 特殊场景处理

### 1. 角色一致性需求
- 建议用户提供角色参考图（@图片1 作为角色形象参考）
- 提醒：暂不支持写实真人脸部素材

### 2. 运镜/动作复刻
- 建议用户提供参考视频（@视频1 参考运镜和动作节奏）
- 说明"完全参考@视频1的所有运镜效果"

### 3. 视频延长
- 明确说明"将@视频1延长Xs"
- 提醒：生成长度应选择"新增部分"的时长

### 4. 剧情颠覆/编辑
- 基于已有视频进行修改时，清晰描述改动点
- 如："颠覆@视频1的剧情，主角从温柔转为狠厉..."

### 5. 音乐卡点
- 建议提供参考视频展示节奏
- 时间轴标注与音乐重拍对齐

## 注意事项提醒

### 平台限制

- ⚠️ **提示词正文不超过 1000 字**（硬性限制，超出会截断或降低生成质量）
- ⚠️ **禁止使用具体人名**（天运子、李白等），用通用描述替代
- ⚠️ **暂不支持上传包含写实真人脸部的素材**（图片和视频均不行），系统会自动拦截
- ⚠️ 视频参考会消耗更多生成额度
- ⚠️ 混合输入总上限是12个文件
- ⚠️ 视频总像素数范围：[409600(640×640,480p), 927408(834×1112,720p)]

### 使用建议

- 💡 建议优先上传对画面或节奏影响最大的素材
- 💡 多模态素材多时，务必检查 @对象标注是否清楚
- 💡 视频延长功能：选择的生成长度应为"新增部分"的时长
- 💡 角色一致性：建议提供多张角色参考图
- 💡 复杂运镜：可用"完全参考@视频1的所有运镜效果"确保精准复刻
